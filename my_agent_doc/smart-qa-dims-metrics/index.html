01 · 指标

算出来的那个"数"

指标是 SELECT 里带聚合函数的字段，最终展示给用户的数值。

基础指标

直接从表的数值字段聚合得来，每个指标有默认聚合方式：

SUM COUNT AVG MAX COUNT DISTINCT
特殊指标 — 维度聚合 特例

对维度字段做聚合拼接，产出文本列表而非数值。

GROUP_CONCAT STRING_AGG COLLECT_SET
派生指标

不能直接从表字段取，分为两类：

① 跨指标公式
客单价 = 销售额 ÷ 下单用户数 毛利率 = (收入 - 成本) ÷ 收入
② 条件聚合（同指标 × 维度值）
线下销售额 = SUM(CASE channel='线下' THEN amount ELSE 0 END) 线上销售额 = SUM(CASE channel='线上' THEN amount ELSE 0 END)
-- 基础指标 + 派生指标（跨指标公式 + 条件聚合） + 维度聚合
SELECT region,
  SUM(amount) AS sales, ← 基础指标
  SUM(amount) / COUNT(DISTINCT user_id) AS arpu, ← 派生：跨指标公式
  SUM(CASE WHEN channel = '线上' THEN amount ELSE 0 END) AS online_sales, ← 派生：条件聚合
  GROUP_CONCAT(product) AS products ← 维度聚合
FROM dw.dwd_order_info
GROUP BY region
指标就是算出来的那个数。分为三类：基础指标——直接从表字段聚合，默认 SUM/COUNT/AVG/MAX；派生指标——不能直接从字段取，包括跨指标公式如客单价=销售额÷用户数，以及条件聚合如线上销售额=SUM(CASE WHEN channel='线上')；维度聚合——对维度字段做 GROUP_CONCAT，产出文本列表。翻下一页看指标案例。

02 · 维度

按什么分组？

维度是数据的"角度"，是 GROUP BY 后面的字段。

定义

非数值型的分组属性字段。用户说"按地区"、"按月份"、"各个品类"，本质就是维度。

常见维度
地区 品类 渠道 门店 用户等级 年份 月份 品牌
-- 按地区、月份维度，统计销售额
SELECT region, month, SUM(amount) AS sales ← 指标
FROM dw.dwd_order_info
GROUP BY region, month ← 维度

拓展问题

维度太多 → 行数爆炸 → LLM 分析不了，怎么办？
① 先聚合到粗粒度 — 只按"省"而非"省×市×区"，减少分组组合数
② 加条件缩小范围 — 通过 WHERE 过滤掉大部分数据，只保留需要分析的局部数据

03 · 过滤条件

不是所有数据都要看

过滤条件对应 SQL 的 WHERE / HAVING，决定"看哪些数据"。

对维度过滤（WHERE）

限定维度的值范围——"只看某几类的数据"

地区 = '华东'
品类 IN ('服装','鞋靴')
年份 >= 2024
对指标过滤（HAVING）

聚合后的结果再筛选——"只看达标的数据"

销售额 > 100 万
订单量 >= 1000
毛利率 < 10%
-- 过滤：只查华东、销售额 > 100 万的品类
SELECT category, SUM(amount) AS sales
FROM dw.dwd_order_info
WHERE region = '华东' ← 过滤维度
GROUP BY category
HAVING SUM(amount) > 1000000 ← 过滤指标
过滤条件对应 SQL 的 WHERE 和 HAVING。WHERE 限定维度值——只看某几类数据；HAVING 对聚合后的指标二次筛选——只看达标的数据。用户说"只看华东""销售额大于100万的"，就是过滤条件。翻下一页看交互案例。

04 · 排序

结果怎么排？

排序对应 SQL 的 ORDER BY，用户常说"Top N"、"从高到低"。

排序方式
DESC从高到低、从大到小
ASC从小到大、从低到高
Top-N取前 N 条，配合 LIMIT
按什么排
按指标排 销售额前 10 名、利润率最高 按维度排 按日期先后、按地区名称
-- 销售额最高的 10 个品类，降序排列
SELECT category, SUM(amount) AS sales
FROM dw.dwd_order_info
GROUP BY category
ORDER BY sales DESC ← 按指标降序
LIMIT 10 ← Top-N
排序对应 SQL 的 ORDER BY 和 LIMIT。用户常说"Top N""从高到低"。可以按指标排——销售额前10名，也可以按维度排——按日期先后。翻下一页看交互演示。

summary

一句问数，拆成四件事

概念 回答的问题 用户怎么说 SQL 对应 举例
指标 要算什么数？ 有多少、卖了多少、占比 SELECT + 聚合 SUM(销售额)
维度 按什么分组？ 各个、按、每种、每个 GROUP BY 地区、月份、品类
过滤 看哪些数据？ 只看、只要、限定在 WHERE / HAVING 地区 = '华东'
排序 结果怎么排？ 前 N 名、从高到低 ORDER BY + LIMIT 按销售额 DESC
-- 一句话："华东地区 2024 年各品类销售额前 5 名"
SELECT category, SUM(amount) AS sales     ↑ 维度↑ 指标
FROM dw.dwd_order_info
WHERE region = '华东' AND year = 2024  ← 过滤条件
GROUP BY category        ← 维度
ORDER BY sales DESC       ← 排序
LIMIT 5              ← 排序 (Top-N)
总结：一句问数，拆成四件事。指标——要算什么数（SELECT+聚合），维度——按什么分组（GROUP BY），过滤条件——看哪些数据（WHERE/HAVING），排序——结果怎么排（ORDER BY+LIMIT）。用户说一句"华东地区各品类销售额前5名"，后端就是这四个概念组合。谢谢大家，有问题欢迎交流。
